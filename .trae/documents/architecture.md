# 慢病用药小管家 - 技术架构与开发规范 (Architecture)

## 1. 技术栈选型
- **前端框架**：React 18 + TypeScript + Vite
- **状态管理**：Zustand (全局状态：用户角色、当前管理对象)
- **路由管理**：React Router DOM v6
- **UI框架/样式**：Tailwind CSS (响应式、组件化样式，便于极速构建)
- **图标库**：Lucide React
- **数据可视化**：Recharts (绘制健康报告与服药依从性图表)
- **日期处理**：date-fns (轻量级，便于处理打卡日期和复诊周期)

## 2. 目录结构设计
```
src/
├── components/          # 复用组件 (如：打卡卡片, 进度条, 缺药警告提示等)
├── pages/               # 页面级组件
│   ├── Home/            # 用药计划与打卡 (集成模块1、2)
│   ├── Inventory/       # 药品库存与补药提醒 (模块3)
│   ├── FollowUp/        # 复诊续方管理 (模块4)
│   └── Report/          # 健康报告与可视化 (模块5)
├── store/               # Zustand 状态 (用户身份管理、模拟数据中心)
├── types/               # TypeScript 接口 (Plan, Medicine, Record, Report)
└── utils/               # 工具函数 (日期格式化、数据聚合)
```

## 3. 核心数据模型 (Mock)
考虑到是快速原型的搭建与演示：
- **User**：包含角色信息(本人/家属)，若为家属则可切换管理的“被看护人”视图。
- **MedicationPlan**：药品名称、用量、频次(如：每日1次/早饭后)、当前库存。
- **MedicationRecord**：打卡记录 (日期、时间段、是否服用)。
- **FollowUp**：上次复诊时间、复诊周期、建议挂号科室。

## 4. 关键交互流程
1. **首页(打卡与计划)**：展示今日早中晚的服药列表，点击“确认服药”直接更新库存和生成记录，显示成功反馈动画。
2. **库存模块**：列表页，计算当前余量，当低于(如7天的用量)时标红并显示“去续方”按钮。
3. **健康报告**：折线图展示近7日/30日的按时服药率，饼图展示漏服/按时服的占比。
