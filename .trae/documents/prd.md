# 慢病用药小管家 - 产品需求文档 (PRD)

## 1. 产品背景
慢性病患者线上复诊开药后，常忘记按时服药或重复购买，缺乏从“问诊－购药－用药－续方”的连贯服务。本产品旨在填补这一空白，建立连贯的患者用药及复诊管理闭环。

## 2. 用户画像及核心痛点
- **中老年慢病患者**：忘记吃药、记不清吃没吃、容易将多种药品搞混。需要最简化的UI和强提醒。
- **上班族慢病患者**：工作忙碌容易漏服，出差时药品带不齐。需要便捷的打卡和库存预警功能。
- **慢病家属（代管）**：远程关心父母是否按时服药。需要权限共享和数据同步查看功能。

## 3. 核心功能模块 (产品结构)
1. **我的用药计划**（核心）：展示每日服药安排，按早、中、晚时间段归类，支持新增、编辑药品及剂量。
2. **服药打卡**（核心）：记录每次服药状态（已服、漏服），提供即时的正向反馈（如打卡成功动画）。
3. **药品库存**（核心）：余量追踪与补药提醒。录入药品总数，打卡后自动扣减，低于阈值触发补药提醒。
4. **复诊续方**（核心）：复诊提醒与续方引导。记录上次就诊时间和复诊周期，提前提醒用户预约挂号或线上续方。
5. **健康报告**（加分）：数据可视化。统计一周/一月的服药依从性（漏服率）、库存消耗曲线等，并支持生成报告分享给家属或医生。

## 4. 交互与体验要求
- **极简设计**：针对中老年人群，字号适当放大，按钮区域易于点击。
- **状态可视化**：用颜色直观区分状态（绿色=已服/正常，红色/橙色=漏服/库存告急）。
- **角色切换**：支持“患者本人”和“家属代管”模式的简单切换。
